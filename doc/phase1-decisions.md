# 第一段 実装判断メモ

*2026-07-16。仕様書の曖昧点について本人に確認した結果。*

1. **付箋のデータ構造**: `parentIds` と `attachedToId` を両方、同じ値で保持する(系譜と付箋関連の二重記録)
2. **テーマの尾**: 第一段はフラット表示のみ(テーマ内部の入れ子生態系は実装しない)
3. **開いた疑問(「?」で始まる粒)**: 種類による優遇を**許可**。温度計算そのものに猶予倍率をかける(死の一法則の明示的例外)
4. **検索**: 「死者の国」という名目は使わず、普通の検索窓とする。alive/drifted/closed を横断検索。蘇生は死んだ粒にのみ表示

- 彗星(帰還日設定)は仕様書の指示通り後回し(未実装)
