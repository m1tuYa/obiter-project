# orbiter — 開発仕様書(第一段: 循環の検証機)

*この文書はClaude Codeへの指示書。同じフォルダに置かれた「思考メモ_ツール概念_最終形.md」(思想と原理)と「思考メモ_周回モデル.md」(視覚・力学モデル)を必ず先に読むこと。本仕様と概念文書が矛盾する場合、概念文書の物理法則が優先される。*

---

## 0. プロジェクトの目的

思考メモツール「orbiter」のプロトタイプ第一段を作る。

- 第一段のゴールは**美しいアプリではなく、動く生態系**。思考の粒が生まれ、沈み、触れば戻り、閉じれば落ちる——この循環が本人の実使用で2〜4週間回るかを検証する
- 見た目は粗くてよい。ただし操作の摩擦は最小でなければならない(摩擦は循環を止める)
- 評価基準は循環の四条件: ①粒が視界から去れる ②戻る道がある ③戻ったとき書ける ④書くたびに系が回る

## 1. 技術方針

- **PCブラウザで動くローカルアプリ**。単一ユーザー、ネットワーク同期なし
- スタックはシンプルに: Vite + vanilla TypeScript(フレームワーク不要)。描画はDOM+CSSで足りる範囲はDOM、粒の空間配置はSVGかCanvasのどちらか実装が単純な方
- **データの永続化は必須**: ブラウザ再起動・PC再起動でデータが消えないこと。localStorageを使う場合はJSONエクスポート/インポート機能を必ず付ける(データ消失はこの検証の致命傷)
- 調律パラメータ(後述)は一つの定数ファイルに集約し、コメント付きで簡単に変更できるようにする

## 2. データモデル

```
Grain(粒) {
  id
  text                 // 本文。一呼吸ぶんの思考
  createdAtWall        // 壁時計の作成時刻(表示用)
  lastTouchEco         // 生態系時刻での最終接触
  status               // alive | drifted(漂流=沈降死) | closed(閉幕)
  closedNote?          // 閉幕時の一言(任意)
  revivedNote?         // 蘇生時の一言(蘇生には必須)
  parentIds[]          // 幹(系譜)。通常0〜1本、合流の粒のみ複数。不変
  attachedToId?        // 付箋の場合、貼り付き先の粒。付箋も実体はGrain
  themeId?             // 析出済みならテーマID
  cometReturnAtWall?   // 自筆の彗星: 帰還予定日(壁時計)。任意
}

Theme(テーマ) {
  id
  name                 // 析出の瞬間に与える名
  createdAtWall
}
```

- **粒は編集可能だが、編集は訂正専用**(誤字直し)。編集しても lastTouchEco は更新しない——時計を巻き戻すのは「手を通した粒の追加」のみ
- 幹(parentIds)は作成時に決まり、以後不変。細い幹(弱い参照)は第一段では実装しない

## 3. 生態系の力学(コア)

### 生態系時刻
- **時間は壁時計ではなく滞在時間で流れる**。アプリがフォアグラウンドでフォーカスされている間だけ、生態系時刻(秒)が進む。不在中は完全凍結
- 実装: ウィンドウのfocus/blur(またはvisibility)で計時を開始/停止し、累積秒を保存する

### 温度と沈降
- 各粒の温度 = f(現在の生態系時刻 − lastTouchEco)。単調減少
- **帯域規制**: 「今」の面に生きて表示される粒の総数には上限がある(調律パラメータ: 初期値 30)。上限を超えたら、最も冷えた粒から status=drifted に落とす
- 加えて、最終接触から一定の生態系時間(調律パラメータ: 初期値 3時間 ※滞在時間ベースなので実日数では数日に相当)を超えた粒も drifted に落とす
- 沈降に警告・通知・確認は一切出さない。静かに消える

### 接触(時計の巻き戻し)
- lastTouchEco を現在に更新する操作は以下のみ:
  1. その粒への付箋の追加(最小単位)
  2. その粒を親とする新しい粒の追加(テーマの先端への追記、合流)
  3. 蘇生(一言を添えて軌道に戻す)
- **閲覧・選択・スクロールは代謝上ゼロ**。読んだだけでは沈む
- 付箋を貼ると、貼られた側の粒の時計が巻き戻る(付箋自身も新しい粒として生まれる)

### 死の二形態
- **漂流(drifted)**: 上記の自動沈降。デフォルトの死
- **閉幕(closed)**: 粒の上でワンクリック(またはワンキー)の印。即時に視界から退場。任意で一言(closedNote)を添えられるが、必須にしない——殺す操作は軽く
- どちらも削除ではない。データは残り、検索で呼べる

### 蘇生
- 死んだ粒(drifted/closed)を検索から見つけて軌道に戻せる。ただし**一言(revivedNote: なぜ戻すか)の入力が必須**——生かす操作は重く
- 蘇生した粒は alive に戻り、lastTouchEco が現在になる

## 4. 操作(動詞は書く系のみ)

実装する操作はこれだけ。**移動・並べ替え・タグ・フォルダ・お気に入け・手動ピン留めは実装しない**:

1. **打ち上げ(書く)**: 常時可能。テキストを打ってEnterで粒が生まれる。デフォルトは無所属
2. **参照系での追記**: 粒またはテーマを選択した状態で書くと、その粒/テーマの先端を親とする粒が生まれる(幹は書いたときの参照系が決める)
3. **付箋**: 粒を選んで短いテキストを貼る。種類の区別(疑問/感情)はUIで分けない——ただの短い粒。「?」で始まる付箋は開いた疑問として視覚的に少し目立ってよい
4. **閉幕**: 粒の上で印一つ
5. **析出**: 粒(通常は付箋)を選んで名前を与え、テーマにする
6. **合流(まとめなおし)**: 複数の生きた粒を選択し、それらを親とする新しい粒を書く
7. **彗星**: 粒に帰還日を設定できる(任意機能。優先度低、後回し可)
8. **蘇生**: 死者の検索結果から、一言を添えて戻す

## 5. ビュー(第一段は3つだけ)

### a. 今の面(デフォルト)
- 生きている粒だけを表示する。周回モデルの簡易版でよい: **角度=接触順、半径=冷え具合**の放射配置。アニメーション不要(静止でよい)。凝った縮退も不要——温度で文字サイズと不透明度を段階的に落とす程度
- 粒は**枠なしのテキスト**として暗い背景に浮かべる(枠・カード・罫線を使わない)
- 開いた疑問付箋(未閉幕の「?」)は沈みにくく見える位置に浮かせる
- テーマに属する粒は、テーマの代表粒(最新)だけを表示し、テーマ名を小さく添える
- 実装が重ければ、初版は「温度順の縦一列(上が熱い、下ほど薄い)」への退化を許す。ただし枠なしの原則は守る

### b. テーマの尾(スレッド)
- テーマを開くと、系譜順の一本の読み物。**最初に見えるのは先端**(最新の粒+開いた付箋)で、遡ると古い方へ
- 先端で書けばテーマへの追記になる

### c. 死者の国(検索のみ)
- 検索窓一つ。alive/drifted/closed を横断して全文検索できる
- 検索結果からの閲覧は代謝ゼロ。そこから蘇生だけができる
- **死者を一覧・ブラウズするUIは作らない**(検索で呼ぶだけ)

## 6. 作ってはいけないもの(第一段の禁止事項)

- 通知、バッジ、未読数、リマインダー(彗星の帰還表示のみ例外)
- 「もうすぐ消えます」等の沈降警告
- 統計・実績・連続記録などのゲーミフィケーション
- タグ、フォルダ、手動並べ替え、ピン留め、お気に入り
- AI機能全般(要約、自動リンク、類似表示)——概念文書で保留中
- 時間駆動の自動再提示(間隔反復)——自筆の彗星のみ合法
- リッチテキスト、画像添付(テキストのみでよい)

## 7. 調律パラメータ(定数ファイルに集約)

- 帯域上限(生きた粒の最大数): 初期値 30
- 沈降までの生態系時間: 初期値 10800秒(滞在3時間)
- 温度→表示(サイズ・不透明度)の段階閾値
- これらは検証期間中に本人が調整する前提。変更が1行で済むこと

## 8. 受け入れ基準

- [ ] 粒を打ち、再起動してもデータが残る
- [ ] アプリを開いている時間だけ粒が冷え、放置した粒が警告なく視界から消える
- [ ] 付箋を貼ると貼られた粒が温まり(視覚的に戻り)、付箋自身も粒として存在する
- [ ] 閉幕がワンアクションで、視界から即時に消える
- [ ] 検索で死んだ粒が見つかり、一言を入力しないと蘇生できない
- [ ] 析出→テーマの尾で先端に追記→合流(複数親の粒)が一通り行える
- [ ] タグ・フォルダ・並べ替えがどこにも存在しない
- [ ] JSONエクスポートでデータを取り出せる
